```
$ npm install -g @openai/codex@0.105.0
```

  View details

## New Features

- The TUI now syntax-highlights fenced code blocks and diffs, adds a `/theme` picker with live preview, and uses better theme-aware diff colors for light and dark terminals. ([#11447](https://github.com/openai/codex/pull/11447), [#12581](https://github.com/openai/codex/pull/12581))
- You can now dictate prompts by holding the spacebar to record and transcribe voice input directly in the TUI. This feature is still under development; to enable it set features.voice\_transcription = true in your config. ([#3381](https://github.com/openai/codex/pull/3381))
- Multi-agent workflows are easier to run and track: `spawn_agents_on_csv` can fan out work from a CSV with built-in progress/ETA, and sub-agents are easier to follow with nicknames, a cleaner picker, and visible child-thread approval prompts. ([#10935](https://github.com/openai/codex/pull/10935), [#12320](https://github.com/openai/codex/pull/12320), [#12327](https://github.com/openai/codex/pull/12327), [#12332](https://github.com/openai/codex/pull/12332), [#12570](https://github.com/openai/codex/pull/12570), [#12767](https://github.com/openai/codex/pull/12767))
- The TUI picked up new convenience commands: `/copy` copies the latest complete assistant reply, while `/clear` and `Ctrl-L` clear the screen without losing thread context, with `/clear` also able to start a fresh chat. ([#12444](https://github.com/openai/codex/pull/12444), [#12520](https://github.com/openai/codex/pull/12520), [#12613](https://github.com/openai/codex/pull/12613), [#12628](https://github.com/openai/codex/pull/12628))
- Approval controls are more flexible: Codex can now ask for extra sandbox permissions for a command, and you can auto-reject specific approval prompt types without turning approvals off entirely. ([#11871](https://github.com/openai/codex/pull/11871), [#12087](https://github.com/openai/codex/pull/12087))
- App-server clients can do more with threads: `thread/list` can search by title, thread status is exposed in read/list responses and notifications, and `thread/resume` returns the latest turn inline so reconnects are less lossy. ([#11776](https://github.com/openai/codex/pull/11776), [#11786](https://github.com/openai/codex/pull/11786), [#12578](https://github.com/openai/codex/pull/12578))

## Bug Fixes

- Long links in the TUI stay clickable when wrapped, which also fixes related clipping and layout issues in several views. ([#12067](https://github.com/openai/codex/pull/12067))
- Several TUI interaction edge cases were fixed: queued-message editing now works in more terminals, follow-up prompts no longer get stuck if you press Enter while a final answer is still streaming, and approval dialogs now respond with the correct request id. ([#12240](https://github.com/openai/codex/pull/12240), [#12569](https://github.com/openai/codex/pull/12569), [#12746](https://github.com/openai/codex/pull/12746))
- `@` parsing in the chat composer is more reliable, so commands like `npx -y @scope/pkg@latest` no longer accidentally open the file picker or block submission. ([#12643](https://github.com/openai/codex/pull/12643))
- App-server websocket handling is more robust: thread listeners survive disconnects, Ctrl-C waits for in-flight turns before restarting, and websocket clients that send `permessage-deflate` can connect successfully. ([#12373](https://github.com/openai/codex/pull/12373), [#12517](https://github.com/openai/codex/pull/12517), [#12629](https://github.com/openai/codex/pull/12629))
- Linux sandboxed commands now get a minimal `/dev`, fixing failures in tools that need entropy or other standard device nodes. ([#12081](https://github.com/openai/codex/pull/12081))
- `js_repl` now reports uncaught kernel failures more clearly, recovers cleanly afterward, and correctly attaches `view_image` results from nested tool calls. ([#12636](https://github.com/openai/codex/pull/12636), [#12725](https://github.com/openai/codex/pull/12725))

## Documentation

- Added a public security policy with Bugcrowd reporting guidance. ([#12193](https://github.com/openai/codex/pull/12193))
- Updated install and local workflow docs to use `cargo install --locked cargo-nextest` and to avoid routine `--all-features` builds unless you specifically need full feature coverage. ([#12377](https://github.com/openai/codex/pull/12377), [#12429](https://github.com/openai/codex/pull/12429))
